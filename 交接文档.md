# 盯盯喵 · 项目交接文档

> 给下一个对话/会话接手用。读完这份就能继续迭代。
> 源码版本 v1.4-dev · 2026-07-14（`dist/`尚未重打，仍是v1.3）

## 一句话
桌面小猫浮窗，用电脑摄像头管你「久坐 / 离座 / 喝水」，纯本地、免费，带托盘、开机自启、气泡按钮和每日健康报告。技术栈 Python + OpenCV + YOLOv4-tiny + tkinter Canvas + pystray。

## 项目位置
`D:\AI\claude code\盯盯喵\`（原名"本地之眼"，v1.2 起改名为盯盯喵）

## v1.4-dev 变了什么（相对v1.3）
- **猫换成图片角色**：桌面端和`promo.html`统一使用用户选定的上一版灰白猫，原始4×2图集拆成8张透明PNG，对应静坐、观察、摇尾、久坐、抱杯、喝水、离座和休息状态
- **图片素材只加载一次**：`cat_sprites.py`负责状态映射、资源路径和点击热区；`Pet`启动时把8张PNG转成`PhotoImage`缓存，重绘时只切换已缓存图片
- **紫边修复**：Windows Tk的`transparentcolor`不支持半透明边缘，显示尺寸上的猫图改用二值alpha，避免半透明像素和`#FE00FE`混合成紫色描边
- **饮水手填可用了**：弹窗由286px增高到376px，确定按钮不再被裁切；支持按钮或Enter提交，非法值会原地提示且不写日志
- **补回真正的侧脸检测**：新增`haarcascade_profileface.xml`，正脸失败后检测原图左侧脸和镜像右侧脸，侧脸参数放宽为`scaleFactor=1.08`、`minNeighbors=4`、`minSize=(60,60)`
- **锁屏立即算离开**：`session_state.py`通过Windows输入桌面判断锁屏；锁屏不吃25秒人脸宽限，立即结束当前在座会话，解锁后必须重新识别人脸才开始新会话
- **默认睡觉，悬停才展开**：在座日常使用`sleep.png`，猫头上只保留无紫边的小型“久坐X”状态签；睡眠姿态会按素材透明留白把状态签额外下移40px，鼠标进入猫后才唤醒并展开完整气泡，气泡和猫组成连续悬停区
- **完整气泡位置优化**：正常坐姿的完整气泡额外上移24px，气泡尾尖不再贴住猫耳；右下角会混成紫色的“右键菜单”文字已彻底删除
- **首次启动教程**：新增三步新手引导，介绍纯本地隐私、悬停/点眼睛/拖动和右键/日志/报告；完成或跳过后按版本记忆，右键与托盘可随时重开
- **教程视觉圆润化**：内容卡片与前后按钮使用圆角无描边绘制，轻阴影替代硬边框；三行网格保证底部导航不被内容卡片挤压
- **设置合并存储**：新增`settings_store.py`原子更新字段，喝水间隔和`onboarding_version`共用`settings.json`时互不覆盖
- **报告标签页图标**：新增`report_icon.py`，从图片猫精灵裁取猫头并生成64×64透明PNG，以Base64内嵌favicon写入健康报告，不再显示浏览器默认地球图标
- **喝水提醒可调**：默认30分钟，可从右键菜单或托盘选择20/30/45/60/90分钟，写入根目录`settings.json`；提醒按最近喝水或最近一次提醒重新起算
- **倾斜杯更容易识别**：杯子置信度由0.4降到0.28；有人脸时只把最大人脸周围区域放大后跑一次YOLO，提升小尺寸、轻微倾斜杯子的占比，同时不增加常驻推理次数
- **端水猫重新修图**：使用内置imagegen精确编辑`hold_cup.png`，移除杯子下方两只多余中央脚，只保留抱杯前爪、圆肚子和两侧后脚
- **动画改成时间驱动**：新增`cat_visual.py`，用`time.monotonic()`采样连续姿态，UI刷新从5FPS提升到约25FPS，动作速度不再依赖某一帧有没有卡住
- **呼吸和尾巴重做**：呼吸使用3.8秒余弦曲线；尾巴改成固定根点的平滑曲线，单次摆动后会停顿，循环接缝不再跳回
- **久坐提醒不再抖窗口**：`over`状态只让猫本体短促侧移、压耳和变表情，不再修改真实窗口坐标
- **喝水动作能看见了**：猫先完成2.4秒举杯动画，再弹饮水记录框；杯子使用连续缓动，前爪跟随杯子移动
- **睁眼展开可见且可取消**：面板使用ease-out曲线，每一步主动重绘，快速反复点击时旧动画会被token终止
- **视频帧缓存**：25FPS重绘时只在摄像头帧对象变化后重新生成`PhotoImage`，避免重复resize
- **监控预览下移与说明**：监控气泡完全展开位置由顶部10px下移到20px；标题改为“低频预览 / 省 CPU”，新手教程同步说明低频抽帧是为了降低CPU占用，并非卡顿
- **猫造型以图集为准**：旧Canvas矢量猫不再由`pet.py`绘制，`cat_visual.py`暂时保留连续动画采样和历史测试，便于后续逐步清理
- **测试**：除历史动画、图片猫、正侧脸和饮水输入外，新增锁屏状态、喝水提醒持久化、悬停UI、色键安全状态签和倾斜杯近脸裁切测试

## 怎么跑
- 源码：双击 `盯盯喵.vbs`
- exe：双击 `dist\盯盯喵.exe`（exe 要和旁边 models/ 在一起）
- 依赖：`pip install opencv-python==4.10.0.84 pillow numpy pystray`

## 坑清单（累加，序号是历史顺序）

1. **opencv 必须 4.10**，5.0 删了 `cv2.CascadeClassifier`（人脸检测）。别升级。
2. **opencv 不认中文路径**：加载 YOLO 模型要先 `shutil.copy` 到系统 temp 英文路径；若要读写图片用 `imencode+tofile` / `imdecode+fromfile`。
3. **打包**：`pyinstaller --onefile --windowed --collect-all cv2 --collect-all PIL --collect-all pystray`，models 不打进 exe、放 exe 旁边（pet.py 用 exe 目录找 models）。**v1.2 加了 pystray 依赖，pyinstaller 命令记得加 `--collect-all pystray`**。
4. **无黑窗**：靠 .vbs `WScript.Shell.Run "...pythonw... pet.py",0` 启动；双击 .bat 会闪 cmd。
5. **验证脚本输出**加 `python -u`，否则强杀进程时 print 缓存丢失、误判没跑。
6. **`-transparentcolor` 的锯齿**：Windows tkinter 用 `wm_attributes("-transparentcolor", "#FE00FE")` 让画布背景透明，颜色必须在画面里绝对不会用到。别在猫身上加接近 `#FE00FE` 的色。
7. **Canvas 圆角矩形靠 create_polygon + smooth=True 近似**，不是真圆角，放大看有一点抖动；够用别追。
8. **画猫用绝对坐标**，`draw_cat(c, cx, cy, ...)` 内部所有坐标相对 head_center。挪位置改调用点就行，不改内部数值。
9. **视频气泡"升起"不是真物理动画**，是画布高度显示区（TRANS 色 hide 矩形）缩小 + 视频 y 位置插值组合出来的。改曲线改 `_animate_h` 的 steps/间隔。
10. **点击热区 hardcoded**：`_release` 里写死了眼睛坐标区间。改猫的位置或大小要同步改热区。
11. **中文 .vbs 编码要 UTF-16 LE with BOM**（Windows wscript 才认得中文路径里的字）。用 `[System.IO.File]::WriteAllText(path, content, [System.Text.Encoding]::Unicode)` 写。**v1.2 起 .vbs 用 WScript.ScriptFullName 自定位**，路径里没有中文字面量了，但 .vbs 本身在中文文件夹里还是需要 UTF-16 编码。
12. **pystray 图标必须是 RGBA PIL Image**，纯色/JPEG/带透明PNG都可以，但要用 PIL 读进来。托盘线程调用 `icon.run()` 是阻塞的，必须放 daemon 线程里。
13. **托盘菜单回调运行在 pystray 线程**，不能直接调 tkinter API（Tk 不是线程安全的）。走 STATE 字典置标志位，让 `Pet.tick` 在 tk 主线程里消费。
14. **自启快捷方式** 用 powershell `WScript.Shell.CreateShortcut` 生成，TargetPath 指向 `.vbs`，不要指向 `pet.py`（那样会闪 cmd）。删除只需 `os.remove(lnk)`。
15. **sit CSV 格式历史断裂**：v1.0/v1.1 没有 sit 日志；v1.2 加了 `start,end,minutes,over`。如果 logs 里有 v1.0 时期的 `sit_*.csv`（列头是 `time,event,sit_minutes`），归档它别让 report 读，否则表格乱。v1.3 起 `read_sit_csv` 判断 header 只认 `start,` 开头。
16. **气泡按钮 hitbox 每次 redraw 后要更新**：`draw_text_bubble` 返回 `{"log":(x1,y1,x2,y2),"report":(...)}`，`Pet.redraw` 存到 `s._btn_boxes`，`Pet._release` 遍历判断。位置每次绘制略变（气泡高度随 nudge 显示切换）。
17. **中文 exe 文件名**：`pyinstaller --name 盯盯喵 pet.py` 会生成 `dist\盯盯喵.exe`。PS 在中文 codepage 下能正常处理，但发布时给别人的电脑注意文件名传输编码（如放 U 盘、上传网盘可能变乱）。
18. **图片猫是外部运行资源**：源码运行从项目根目录读取`assets/cat_sprites/`；打包后`cat_sprites.asset_root()`从exe同级读取，所以重打`dist/`时要把整个`assets/cat_sprites/`复制到exe旁边。`cat_sprite_source.png`和拆分脚本不需要随发布包分发。
19. **透明色窗口不能保留PNG半透明边缘**：Tk会先把半透明像素和Canvas的`#FE00FE`混合，Windows再按精确色键透明，结果就是紫边。必须在最终显示尺寸上把alpha二值化，不能只检查原始PNG四角透明。
20. **无边框弹窗固定高度必须做真实布局检查**：`pack()`内容超出固定geometry时不会自动扩窗，底部按钮会直接裁掉。饮水框当前为330×376，按钮底部实测371px；修改字体、间距或文案后要重测。
21. **“alt”不是侧脸分类器**：`haarcascade_frontalface_alt.xml`仍然是正脸模型。侧脸必须另载`haarcascade_profileface.xml`，并对水平镜像再跑一次才能覆盖两个朝向。
22. **锁屏不能继续走人脸宽限期**：锁屏时最后一次人脸时间仍很新，如果只把它当一次漏检，会继续算在座。锁屏分支必须在摄像头读取和宽限判断之前执行，并清空`last_seen`。
23. **悬停气泡必须让猫和气泡共用命中区**：只检测猫本体会导致鼠标一移向日志/报告按钮，气泡立刻消失。展开后使用从气泡顶到猫底的连续区域维持显示。
24. **透明Canvas上的文字也会出现紫边**：不只是PNG，Tk文字抗锯齿也会和`#FE00FE`混色。简洁状态使用缓存的Pillow不透明小图，alpha仍只允许0或255。
25. **喝水提醒不能在到期后每分钟重复**：到期判断必须以`max(最近喝水, 最近提醒)`为锚点；用户改间隔后持久化到`settings.json`，首次无饮水记录则从本次启动时间起算。
26. **不要把COCO第40类`wine glass`用于摄像头饮水识别**：眼镜近景很容易被误判成高脚杯。当前只保留`bottle`和`cup`，再用`is_cup_near_face()`排除眼睛区域、扁宽框和离嘴太远的候选；改阈值时必须同时跑眼镜与倾斜杯测试。
27. **共享`settings.json`不能整文件覆盖**：喝水间隔与教程版本属于不同功能，写入时必须通过`settings_store.update_settings()`合并现有字段并原子替换；否则改一次提醒间隔就会让新手教程再次弹出。
28. **报告favicon必须内嵌**：健康报告是可单独移动的自包含HTML，不能引用项目目录下的相对图片路径；使用`report_icon.report_favicon_link()`生成Base64 PNG标签，并保留资源缺失时的猫头回退图。
29. **低频监控预览不能标成LIVE**：检测线程按`CHECK_EVERY`抽帧以控制CPU占用，监控画面不是连续视频；UI应明确写“低频预览 / 省 CPU”，教程说明“并非卡顿”，避免用户误判性能问题。

## 检测逻辑要点（在 pet.py 顶部参数）
- 判在座：先用两个正脸分类器（default+alt，minNeighbors=5，minSize=80）；都失败后用profile分类器检查原图和水平镜像（minNeighbors=4，minSize=60），覆盖左右侧脸；再叠加`GRACE_SEC=25`宽限期，`over`状态使用`OVER_GRACE=5`
- 遮挡：画面平均亮度 < `BLOCK_LEVEL=22` 判被捂
- 喝水：YOLO只检测bottle/cup（class 39/41），候选阈值0.28；选择最大人脸周围3.2倍脸宽、2.8倍脸高的区域放大到416×416检测，再还原全图坐标；`is_cup_near_face(f,c)`排除眼镜等眼睛区域扁框，连续2帧命中继续负责防误报
- 摄像头**常开**（cap 不每轮开关）；采样间隔 `CHECK_EVERY=0.5s`；帧缩略图 200×150 存 `STATE["frame"]`
- 会话结束（away/blocked 从 seated/over 切换）时写 slog_write，短于 0.5min 的段忽略

## UI 内部结构
- `Pet`类：tk.Tk overrideredirect + 单Canvas。当前窗口宽236、`H_CLOSED=250`、`H_OPEN=410`、气泡宽210，通过`_cur_h`控制视觉显示高度，顶部超出部分用TRANS色矩形盖掉营造展开效果。窗口位置按最大高度定位不动。
- 事件：`<ButtonPress/B1-Motion/Release-1>`区分拖窗和点击；普通`<Motion>/<Leave>`控制完整气泡，仅猫和已展开气泡的联合区域保持显示；右键菜单和托盘都可打开新手教程。
- 绘制：`cat_sprites.select_sprite()`把业务状态映射到8张透明PNG，`Pet`使用缓存的`ImageTk.PhotoImage`绘制144×144图片猫；`pet.py`保留`draw_text_bubble`、`draw_video_bubble`和`rr()`。`redraw()`约每40ms全清重画，视频图像只在检测帧更新时转换一次；监控框完全展开停在顶部20px。
- 动画：正常在座且无提醒时默认显示用户指定的蜷睡小猫（源图`assets/cat_sleep_source.png`）；悬停时唤醒并展开完整气泡。久坐、抱杯、喝水、离座等业务状态仍优先于休息姿态，喝水动画2.4秒后再弹记录框。
- 托盘 & tk 通信：`STATE["_toggle_eyes"]`, `STATE["_quit"]`, `STATE["paused"]` 三个标志位。托盘线程写，tk tick 读+消费。
