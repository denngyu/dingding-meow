# 盯盯喵项目规范

## 项目目标

这是一个 Windows 桌面猫应用。改动优先保证：视觉自然、动画流畅、交互不打扰、资源占用可控、现有启动方式兼容。

## 开工前三道关卡

1. 先读取本文件、`交接文档.md`、`项目复盘.md`，再查看代码。
2. 先确认当前启动入口、打包方式和已有版本存档，不直接修改 `dist/`、`dist_v1old/` 或 `版本存档/` 中的产物。
3. 修改前记录现状与可验证目标；修改后运行针对性测试，并实际启动检查视觉效果。

## 目录与修改边界

- 源码入口以根目录 `pet.py` 为准，除非代码检查证明另有入口。
- `dist/`、`dist_v1old/`、`版本存档/` 视为构建产物或历史版本，只读参考，不手工修改。
- 日志写入 `logs/`；任务工作记录写入 `工作记录/`。
- 不读取或输出任何密钥、令牌、密码。
- 若未来出现业务数据，只允许读取 `项目/<品牌>/data/masked/`；禁止读取 raw、已脱敏、敏感词表和脱敏映射文件。

## 设计与动画约定

- 动画首先服务于猫的生命感：动作应有准备、发力、缓冲和停顿，避免匀速机械位移。
- 保持轮廓清晰，避免装饰堆叠、过度弹跳、持续晃动和无意义粒子效果。
- 空闲动画低频、低幅；交互反馈短而明确；移动动画需要方向感和落脚节奏。
- 所有循环动画应避免明显跳帧，并考虑不同刷新率和系统缩放。
- 默认尊重性能：避免无界定时器、重复创建图形对象和高频磁盘 I/O。

## 工程纪律

- 找根因，不用注释报错或添加绕过标记让程序表面可运行。
- 非平凡改动先写计划和验收标准；功能或修复优先补可失败的测试，再实现。
- 保留用户已有改动，不重置、不覆盖无关文件。
- 文件编辑使用精确补丁；构建产物由构建命令生成。
- 修改完成后至少执行语法检查、相关自动化测试和一次启动级验证。

## 子代理分工

- 主体负责意图、设计判断、最终整合与验收。
- 只有在工作包可独立并行时才委派；同一文件的紧耦合修改不拆分。
- 子代理默认只做分析或单一边界明确的工作，不同时改同一文件。

## 错误记录

- 一般踩坑补充到本文件的“已知坑”章节。
- 严重错误记录到 `error_log.md`，包含现象、根因、影响、修复和预防措施。

## 已知坑

- 项目曾缺失 `AGENTS.md` 和 `CLAUDE.md`，导致规范映射失效；后续不得删除其中任一文件。
- 4×2 或多行动画图集的每一格即使尺寸一致，主体中心和脚底基线也可能不同；直接逐帧播放会让角色上下跳。拆帧后必须按非透明主体包围盒统一水平中心与脚底锚点，并用测试比较各帧偏差。
- 伸掌这类轮廓变化很大的动作不能用整张非透明包围盒当稳定锚点：手掌变大时包围盒中心会移动，反而拖着身体晃。应固定脸部或躯干锚点，并只在窗口坐标真正变化时提交`geometry()`。
- 图集里的角色若越过单元格边缘，相邻格会混入尾巴或耳朵碎片；拆帧后应先保留最大透明连通域，再做主体对齐。
- 图集主体也可能主动跨出自己的单元格，严格按网格裁剪会留下长直线断面；宽睡姿等贴边角色要检查 alpha 包围盒边缘的连续像素，并为对应单元格保留安全重叠区。
- 生图角色的肢体数量不能靠“图片完整、单连通体”测试代替视觉验收；抱杯姿态要固定检查杯底中央轮廓，确保只有两只抱杯前爪和两侧后脚，不出现重复中央腿。
- 悬停唤醒不能只用布尔值直接切睡姿/坐姿；需要一次性动作状态、开始时间和取消条件，且鼠标在气泡联合命中区内移动时不能反复重启。
- Windows 分层透明 Tk 窗口不能用 `CopyFromScreen` 可靠验收：它可能漏掉窗口本身只截到后景，也可能把项目外桌面内容带入截图。桌面验收默认运行专用演示脚本并观察退出结果；需要可留档画面时只渲染离线 Canvas，禁止截真实桌面。
- 人脸检测宽限与久坐会话续接是两层规则：短暂遮挡可在进入离开状态后保留原起点 30 秒，但锁屏必须立即结束会话，不能借续接窗口继续计时。
- Windows `OpenInputDesktop()` 返回句柄不等于未锁屏；必须再调用 `SwitchDesktop()`，无法切换时按锁屏处理，并在 `finally` 中关闭句柄。
- 摄像头 `read()` 失败不能直接重连并保留旧 UI 状态；无帧期间也必须推进离开计时，超过 30 秒结束久坐会话，否则锁屏或摄像头暂停会把“久坐”永久冻在界面上。
- Haar 人脸检测不能采用“任一分类器命中即算脸”；正脸应由 default/alt 重叠确认，左右侧脸应过滤原图与镜像落在同一区域的对称假目标，避免柜体纹理长期冒充人脸。
