# v1.5 首轮迭代：延时摄影、饮水降误报、圆角弹窗、换肤与摄像头恢复

## 本轮目标

- 增加显式开启的纯本地延时摄影。
- 收敛垃圾桶被识别成杯子后触发饮水弹窗的问题。
- 让饮水记录框更圆润，喝水提醒时端水猫停留 3 分钟。
- 增加可扩展皮肤系统，并提供银灰经典、暖橘元气两套日常状态。
- 修复电脑休眠或锁屏后，摄像头句柄失效且无法恢复的问题。

## 已实现

- `timelapse.py`：默认每 60 秒取一帧，停止或退出时在
  `logs/timelapse/` 合成 MP4；默认关闭，只有用户从右键或托盘手动开始才录制。
- 延时摄影在锁屏、离开、遮挡和暂停检测时不取帧；停止后气泡短暂提示已保存。
- 右键和托盘增加 `查看延时摄影`，只打开 `logs/timelapse/`，不切换录制状态；
  气泡按用户反馈恢复为日志、报告两个按钮。
- `cup_detection.py`：加入 YOLO objectness 下限、候选尺寸/宽高比限制、
  杯口与下脸重叠约束；连续命中次数从 2 次提高到 3 次。
- 饮水框改为透明色键圆角外壳，快捷量卡片、关闭按钮和确定按钮都使用圆角 Canvas。
- 快捷饮水量删除“一瓶 500ml”，新增“一大口 100ml”；当前为 50 / 100 / 150 / 300ml。
- 饮水框高度由 392px 增至 420px，补足确认按钮下方的快捷键提示与安全底距；
  标题区、空白背景和底部提示支持拖动无边框窗口。
- 自动提醒弹出饮水框时，等待窗口显示后强制聚焦 ml 输入框，并在 120ms 后确认一次焦点。
- 新手教程窗口显式加载盯盯喵 `dingdingmeow.ico`，替换 Tk 默认羽毛图标。
- 喝水提醒端水状态由 15 秒延长为 180 秒；只延长视觉陪伴，不重复弹饮水记录框。
- `skin_system.py`：自动发现完整皮肤包，写入共享 `settings.json` 时保留其他字段。
- 使用 imagegen 以经典 4×2 图集为参考生成暖橘皮肤，并拆分为 8 个透明运行状态。
- 中央叩屏动作皮肤的首次 imagegen 编辑未保住透明通道和原尺寸，未接入不合格素材；
  当前中央动作继续回退经典素材，日常 8 状态完整跟随皮肤。
- `camera_capture.py`：锁屏立即释放摄像头；解锁后等待 1.5 秒再打开；
  DirectShow 失败时回退 Windows 默认后端，失败后持续轮换重试。
- 检测循环增加 supervisor 与 3 秒 UI 心跳保护，线程异常不再冻结旧久坐状态。
- 橘猫运行时素材增加“只保留最大连通主轮廓”的防护，移除 `away.png` 右侧串入的邻格碎片。
- 摄像头读取连续失败或返回连续纯黑无信号帧后进入 `camera_off`，释放摄像头并显示
  `点击开始检测`；普通暗场仍保留“镜头被挡住了”。

## 验证

- Python 3.12 语法检查通过。
- 真实摄像头释放/重连验收通过：重连前后均获取到 640×480 画面，未保存帧。
- 离开后仍显示久坐的问题尚未完成有效现场复现；此前诊断期间用户仍在座位前，
  因此“空位家具误报”数据无效，基于该数据的活体过滤已全部撤销。
- `工具/preview_v15_features.py` 已真实启动，展示暖橘皮肤与新圆角饮水框后自动退出。
