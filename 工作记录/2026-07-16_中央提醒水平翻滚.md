# 中央提醒水平翻滚

## 修改前现状

- 中央提醒启动时窗口直接跳到屏幕中央，叩屏完成后直接跳回原位。
- 中央目标同时重算 X、Y，猫会离开用户原本放置的垂直位置。
- 当前只有叩屏动作，没有去程和回程的移动过渡。

## 目标与验收标准

- 去程与回程只修改窗口 X 坐标，Y 坐标从开始到结束保持完全不变。
- 猫用约 3.2 秒慢慢翻滚到屏幕水平中央，提醒完成后以相同时长翻滚回原位。
- 翻滚角度与水平位移使用同一缓动进度，到达中央和回到原位时保持正向，避免切换叩屏图时跳帧。
- 翻滚期间在猫身两侧绘制少量、确定性的生气蒸汽；不创建持续增长的粒子对象。
- 旋转图片在启动阶段一次性缓存，重绘阶段只切换缓存帧，不做高频图片旋转或磁盘读取。
- 三次叩屏和最终第一帧停留保持不变。
- 通过语法检查、完整自动化测试和专用启动演示。

## 实现结果

- 时间轴改为 `roll-in → knock → hold → roll-back`，总时长 13.25 秒。
- 去程和回程各 3.2 秒，水平位移与旋转角度共用 smoothstep 缓动。
- 完整旋转圈数按水平距离取整，保证到达中央和返回原位时猫保持正向。
- `cat_center_target()` 只计算目标 X，目标 Y 直接沿用启动时窗口 Y。
- Tk 窗口移动统一经过 `_set_center_nudge_window_position()`；该函数强制使用原始 Y，仅在整数坐标变化时提交 `geometry()`。
- 新增独立素材 `assets/cat_animations/roll/roll_ball.png`：灰白猫背身蜷成一团，只保留耳尖、尾巴和条纹，不显示眼睛、嘴或完整四肢。
- 从独立毛球素材预生成 24 张旋转帧，统一二值 alpha，并计算透明留白造成的底部对齐补偿；翻滚不再复用叩屏第一帧。
- 毛球显示直径由标准猫尺寸的 86% 继续下调到 56%，运行时约 81px，叩屏阶段仍保持原猫尺寸。
- 蒸汽改为左右各一组小型三瓣云团，使用较浅的暖灰色，并随翻滚轻微上飘、左右摆动；强度曲线在去程和回程两端缓慢淡入淡出，避免突然出现或消失。仍不使用粒子集合、随机数或额外定时器。
- 删除玻璃凹陷与扩散波纹：时间轴不再计算压力值，Canvas 不再绘制玻璃反馈，只保留三次贴掌叩击。
- 专用演示时长改为跟随 `center_nudge.TOTAL_DURATION`，避免回程尚未结束就退出。

## 素材生成

- 模式：参考现有叩屏猫配色与插画风格，重新生成独立位图。
- 提示重点：严格沿用原猫的粗深描边、平涂色块和简化条纹，背身/侧背蜷缩成近圆形毛团；不显示眼睛、嘴和正脸，不使用写实毛发或 3D 质感；只保留灰白配色、三道条纹、耳尖与尾巴暗示。
- 生成后移除纯绿底并保留透明 PNG，程序显示前再转为 Windows 色键安全的二值 alpha。

## 验证

- 语法检查通过：`center_nudge.py`、`cat_sprites.py`、`pet.py`、`工具/preview_stable_startup.py`。
- 自动化测试：97 项全部通过。
- 25FPS 离线采样阶段顺序正确，Y 坐标集合只有 `{640}`。
- 示例 758px 水平路程的单帧最大 X 位移为 15px。
- 旋转帧全部为非空 RGBA 图，alpha 仅包含 0/255。
- 真实 Tk 演示完成去程、叩屏、停留与回程，并正常自动退出。
