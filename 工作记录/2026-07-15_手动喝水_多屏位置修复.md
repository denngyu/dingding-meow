# 2026-07-15 · 手动喝水记录 + 多屏切单屏位置守护

## 需求

1. **手动记录喝水**：用户不在电脑前喝水时，摄像头识别不到；右键菜单和托盘菜单要能手动触发喝水记录弹窗。
2. **多屏切单屏，猫回主屏右下角**：副屏拔了之后小猫仍留在副屏坐标（等于不可见），需要自动 snap 回主屏可见右下角。

## 现状

- 喝水弹窗触发：只在 `loop()` 里 `is_cup_near_face` 命中 → `STATE["drink"]=True` → `tick()` → `_request` → `_show_water_dialog`。
- 位置初始化：`Pet.__init__` 用 `winfo_screenwidth/height` 算主屏右下角，之后不再检查。用户拖到副屏、副屏拔了，窗口留在虚拟屏外。

## 验收标准

### AC1 · 手动记录喝水
- **AC1.1** 右键菜单出现 "手动记录喝水" 项，点它立刻弹出 `WaterDialog`（不吃 `DRINK_ANIM_SEC` 延迟）
- **AC1.2** 托盘菜单出现 "手动记录喝水" 项，从 pystray 线程点它，主线程接管弹出 `WaterDialog`
- **AC1.3** 用户在弹窗选择或手填 ml → 写入 `water_YYYYMMDD.csv`，更新 `STATE["water"]`、`last_drink_ts`、`water_reminder_anchor`（复用 `_show_water_dialog` 现有逻辑）
- **AC1.4** 用户取消弹窗 → 不写日志、不动累计
- **AC1.5** 已经有一个弹窗待开（`_drink_dialog_pending=True`），再次触发不叠开第二个
- **AC1.6** 摄像头检测触发的自动弹窗行为不变

### AC2 · 多屏位置守护
- **AC2.1** 窗口中心点落在任何一块当前显示器上 → 不动
- **AC2.2** 窗口中心点不在任何显示器上（副屏拔了 / 分辨率缩小）→ snap 回主屏右下角
- **AC2.3** 巡检节流：每 2 秒最多一次，不打扰
- **AC2.4** 用户主动拖动过程中（`_dragging=True`）不 snap
- **AC2.5** 非 Windows 平台或 `MonitorFromPoint` 不可用时默认认为窗口可见，不误 snap

## 实现

- 新增 `window_placement.py`：`is_point_on_monitor(x,y)`（Windows `user32.MonitorFromPoint` + `MONITOR_DEFAULTTONULL`），`snap_target(sw,sh,ww,wh)` 算主屏右下角，`needs_reposition(cx,cy,ww,wh,on_monitor_fn)` 用注入检查器判断中心点是否可见。
- 新增 `tests/test_window_placement.py`：4 个纯函数测试，注入假 `on_monitor_fn` 覆盖真假两条路径。
- `pet.py`:
  - `Pet.__init__`: `s._last_screen_check=0.0`；`STATE` 加 `"_manual_drink":False`。
  - 抽小方法 `_request_water_dialog(immediate=False)`，把 `tick` 里已有的 `STATE["drink"]` 分支和新的 manual 触发都收拢过来，`_drink_dialog_pending` 做去重。
  - `tick()` 每 2 秒调 `window_placement.needs_reposition`；若 `_dragging` 则跳过。
  - `_menu_close` 顶部加 `"手动记录喝水"` 项。
  - `start_tray` 菜单加 `"手动记录喝水"` 项，通过 `STATE["_manual_drink"]=True` 跨线程通信。

## 验证

1. `python -m py_compile pet.py window_placement.py` 语法过
2. `python -m pytest tests/test_window_placement.py -q` 通过
3. `pythonw pet.py test` 起来后：右键菜单点"手动记录喝水" → 弹窗立即出现 → 填数字确定 → 关掉猫 → `logs/water_YYYYMMDD.csv` 追加了一行
4. 托盘右键菜单同样点"手动记录喝水" → 弹窗出现
5. 位置检查：手动 `s.r.geometry("+9999+9999")` 强制推到屏外 → 2 秒内看到猫弹回主屏右下角

## 不做

- 不做用户设定"每日饮水目标 ml"面板（超出本次需求）
- 不做位置持久化（拖动位置存盘、下次启动读）——单开工作单
- 不重打 exe / 不发新 release（等用户确认后再走 v1.0.1 流程）
